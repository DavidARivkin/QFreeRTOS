74 /*
75  * Saves the stack pointer for one task into its TCB, calls

88 /* There are slightly different versions depending on whether you are building
89 to include debugger information. If debugger information is used then there
90 are a couple of extra bytes left of the ISR stack (presumably for use by the
91 debugger). The true stack pointer is then stored in the bp register. We add
92 2 to the stack pointer to remove the extra bytes before we restore our context. */
